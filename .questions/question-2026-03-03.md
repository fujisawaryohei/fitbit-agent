# 技術Q&A - 2026-03-03

### `@MapperScan` の動作について

`@MapperScan("com.fitbitagent.repository")` は、指定したパッケージ配下のインターフェースを MyBatis Mapper として自動登録するアノテーション。

- 指定パッケージ内のインターフェースをスキャンし、Spring の DI コンテナに Bean として登録する
- MyBatis が実装クラスを自動生成するため、インターフェースを定義するだけで `userMapper.findById(1L)` のように呼び出せる
- `@MapperScan` を Config クラスに定義している場合、各 Mapper インターフェースに個別に `@Mapper` を付ける必要はない
- `@Service` 等から `@Autowired` や `@RequiredArgsConstructor` でインジェクション可能になる

### `@Configuration` クラスが何も継承・実装しない理由

Spring では継承ではなくアノテーションで役割を伝える設計のため、`@Configuration` を付けるだけで設定クラスとして認識される。

- Spring Boot 起動時に `@Configuration` 付きクラスを自動スキャンする
- 継承が必要だったのは旧来の Spring MVC の書き方（`WebMvcConfigurerAdapter` 等）
- `@MapperScan` はクラスに付けるだけで効果があるため、クラス本体は空でよい
- `@Bean` メソッドを追加すれば手動で Bean 定義も可能

### TokenEncryptionService で暗号化ライブラリを使わない理由

AES-256-GCM は Java JDK に標準で組み込まれている（`javax.crypto`）ため、外部ライブラリ不要。

- JDK標準: 追加依存なし、AES-256-GCMで十分な安全性
- Bouncy Castle: 特殊な暗号方式が必要な場合に使用
- Jasypt: application.yml の設定値を暗号化したい場合に有用
- AWS KMS: 本番環境で暗号化キーをサーバーに置きたくない場合に採用を検討
- 今回の用途（OAuthトークンの暗号化）には標準ライブラリで十分なためシンプルな構成を選択

### `@MapperScan` は `@Configuration` クラスに付ける必要があるか

必須ではない。`@SpringBootApplication` が付いたメインクラスに直接付けることもできる。

- `@MapperScan` が動作する条件は「Spring に管理されているクラス（`@Component` 系）に付いていること」だけ
- メインクラスに付けるとシンプルだが、設定が増えると肥大化しやすい
- 専用 Config クラスに分けると MyBatis 関連の設定を1ファイルに集約でき、拡張性が高い
