# 技術Q&A 2026-03-09

### OAuth2.0コールバックのレスポンス形式について

認可URL押下後、Fitbitの認可処理が完了してアプリにリダイレクトされる際、レスポンスステータスは302で、レスポンスボディに認可コードが含まれるか？

302リダイレクトという認識は正しい。ただし、認可コードはレスポンスボディではなく、リダイレクト先URLのクエリパラメータに含まれる。

例: `GET /api/auth/callback?code=XXXXX&state=YYYYY`

Springでは `@RequestParam String code` で受け取る。レスポンスボディは空。

### OAuth2.0の `state` パラメータについて

`buildAuthorizationUrl(String state)` の引数 `state` は何を指すか？

OAuth2.0のCSRF対策用のランダム文字列。ログイン時にサーバーがUUIDを生成してセッションに保存し、認可URLのクエリパラメータに付与してFitbitに送る。Fitbitはコールバック時にそのまま返してくるので、セッションに保存した値と照合することで「自分のアプリが開始したフローか」を検証できる。一致しない場合は攻撃の可能性があるためエラーにする。

### `@Slf4j` とは何か

Lombokが提供するログ出力用アノテーション。クラスに付けると `private static final Logger log = LoggerFactory.getLogger(...)` というフィールドを自動生成してくれる。`log.info(...)` / `log.warn(...)` / `log.error(...)` などで使用する。`System.out.println` の本番用代替品で、ログレベルによる出力制御やファイル書き出しが可能。

### `@Slf4j` のログ出力の役割分担

Lombokが提供するのはあくまで `log` フィールドの自動生成のみで、ログ出力の実装は別のパッケージが持つか？

正しい。役割分担は3層構造になっている。Lombokは `log` フィールドを生成するだけ。SLF4Jが `log.info()` などのAPIインターフェースを定義。Logback（Spring Bootデフォルト内蔵）が実際にコンソール・ファイルへ書き出す実装を持つ。

### SLF4J はSpring専用か、Java共通か

SLF4J はJava共通の独立したライブラリ。Spring専用ではない。Javaのロギングライブラリが乱立していた歴史的背景から、どのロギング実装（Logback / Log4j2 / java.util.logging等）を使っても同じコードで書けるようにするための統一APIとして作られた。Spring BootはデフォルトでLogbackを採用しているが、SLF4J自体はSpringとは独立して存在する。
