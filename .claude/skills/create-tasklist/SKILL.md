# タスク一覧作成スキル

あなたはPM（プロジェクトマネージャー）として、ユーザーと対話しながらタスク一覧を作成します。

**タスクの粒度の目安**: 1タスク = 変更ファイル数 10〜15以内。各タスクが1エージェントに割り当て可能な単位になるよう意識する。

## 事前準備

以下のファイルが存在すれば読み込む:
- `docs/requirements.md` — FR-xxx の機能要件を参照
- `docs/basic-functional-design.md` — 機能一覧・画面一覧を参照
- `.steering/tasklist.md` — 既存ファイルがあれば加筆モード

## 進め方

**1ステップずつ**進める。

### Step 1: プロジェクト名・フェーズ構成の確認

- プロジェクト名（`# TODO: {プロジェクト名}` に使用）
- 実装をどのようなフェーズに分けますか？
  （例: Phase 0: セットアップ / Phase 1: 認証 / Phase 2: コア機能 等）
- フェーズ数と各フェーズの名前・目的を確認する

### Step 2: タスクの洗い出し（フェーズごと）

各フェーズについて、以下を確認する:
- このフェーズに含まれるタスクを列挙する
- 各タスクについて:
  1. **タスク名**（例: 「Spring Boot プロジェクト初期化」）
  2. **概要**（1〜2文で何をするか）
  3. **機能ID**（対応する FR-xxx / SCR-xxx。なければ `-`）
  4. **優先度**（高 / 中 / 低）
  5. **受入条件**（テスト可能な完了基準を2〜4個）
  6. **ブロック依存**（このタスクを始める前に完了が必要な TASK-xxx。なければなし）
  7. **影響ファイル・ディレクトリ**（変更予定のパス。ファイル競合チェックに使用）

### Step 3: 並列グループの編成

洗い出したタスクを「同時に実行できるグループ」と「直列で進めるべきタスク」に整理する。

**並列実行の判断基準**:
- 影響ファイル・ディレクトリが他のタスクと重複していない
- ブロック依存が解消済み（または依存タスクと同時に実行する必要がない）

上記を両方満たすタスクは並列実行可能。以下の形式で整理してユーザーに提示する:

```
【並列グループ A】同時実行可能
  - TASK-xxx: ...
  - TASK-yyy: ...

【直列】グループ A の完了後
  - TASK-zzz: ...

【並列グループ B】同時実行可能
  - TASK-aaa: ...
  - TASK-bbb: ...
```

### Step 4: TASK-IDの採番

- TASK-xxx の番号体系を確認する
  （例: Phase 0 → TASK-001〜, Phase 1 → TASK-010〜, Phase 2 → TASK-020〜 等）
- 各タスクに採番する

### Step 5: 全体レビュー

- 洗い出したタスク一覧をフェーズ別に提示する
- 並列グループの図を提示し、並列化の方針をユーザーと確認する
- 漏れているタスクや優先度の調整を確認する
- ユーザーの承認後に保存する

## 出力

`template.md` の構成でタスク一覧を生成し、`.steering/tasklist.md` として保存する。
各タスクに `parallel-group` フィールドを追記する（例: `parallel-group: A`、直列の場合は `-`）。
末尾の更新履歴に今日の日付と「初版作成」を追記する。
