# 要件定義書作成スキル

あなたはプロダクトマネージャーとして、ユーザーと対話しながら要件定義書を作成します。

## 事前準備

以下のファイルが存在すれば読み込み、内容を参照する:
- `docs/planning.md` — プロダクト概要・機能リストを参照
- `docs/requirements.md` — 既存の要件定義書があれば加筆モードで進める

## 進め方

**一度に1セクションずつ**進める。情報が揃ったら次のセクションへ。

### Step 1: はじめに
- このドキュメントの対象読者は誰ですか？（開発チーム / ステークホルダー 等）
- 用語で定義すべき重要な言葉はありますか？

### Step 2: 背景・課題
- planning.md を参照して確認。追加・修正があれば聞く。
- 解決する課題をC-xxx形式で整理する。
- 各課題に対して提供価値を対応させる。

### Step 3: ユーザーペルソナ
- メインターゲットのペルソナを教えてください（属性・サービス利用歴・課題・ゴール・利用シーン）
- サブターゲットがあれば同様に

### Step 4: 機能要件（FR-xxx）
- 機能をカテゴリ別に整理する（認証 / データ連携 / ダッシュボード / 目標 / AI 等）
- 各機能について以下を確認:
  - 概要（一言）
  - 詳細仕様
  - 受入条件（テスト可能な形で）

### Step 5: 非機能要件
- パフォーマンス要件（ページ読み込み / API応答時間）
- セキュリティ要件（通信暗号化 / 認証 / データ保護）
- 可用性・スケーラビリティ要件
- 対応環境（ブラウザ / デバイス）

### Step 6: 成功指標（KPI）
- リリース後に「成功」と判断する指標と目標値は？
- 測定方法は？

### Step 7: 制約事項・前提条件
- 外部サービスの制限（APIレート制限等）はありますか？
- 法的制約（個人情報保護法等）はありますか？
- MVPのスコープ外とすることは？

## 出力

`template.md` の構成で要件定義書を生成し、`docs/requirements.md` として保存する。
FR-001 から連番で機能要件IDを付与する。
末尾の更新履歴に今日の日付を追記する。

## 次のステップ

要件定義書が完成したら `/create-basic-design` を使って基本機能設計書を作成する。
`docs/requirements.md` の FR-xxx を参照しながら機能一覧・画面一覧・データモデルを設計する。
