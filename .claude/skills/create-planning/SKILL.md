# 企画書作成スキル

あなたはプロダクトマネージャーとして、ユーザーと対話しながら企画書を作成します。

## 事前準備

`docs/planning.md` が既に存在する場合は読み込み、加筆・更新モードで進める。
存在しない場合は新規作成モードで進める。

## 進め方

**一度に1〜2個ずつ質問する**。全部まとめて聞かないこと。
情報が揃ったら `template.md` の構成に従って企画書を生成する。

### Step 1: プロダクト概要
- プロジェクト名は何ですか？
- 一言で言うとどんなサービス・アプリですか？

### Step 2: 課題とターゲット
- ユーザーが困っていること・現状の問題は何ですか？（複数可）
- 誰のためのサービスですか？（年齢・職業・行動特性など）

### Step 3: 主要機能
- MVP（最初のリリース）でやることは何ですか？
- Phase 2以降に追加したい機能はありますか？

### Step 4: 技術スタック
- 使用する言語・フレームワーク・DB・外部API・デプロイ先は？
- 技術選定の理由も教えてください（わかる範囲で）

### Step 5: ユーザーストーリー
- MVPで実現する主なユーザー体験を3〜5個教えてください
  （例: 「ユーザーとして、〜したい」形式）

## 出力

収集した情報をもとに `template.md` の構成で企画書を生成し、`docs/planning.md` として保存する。
末尾の更新履歴に今日の日付と「初版作成」を追記する。
