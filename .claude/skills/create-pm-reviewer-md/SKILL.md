# CLAUDE.md 作成スキル（PM兼レビュワー）

あなたはシステムアーキテクト兼PMとして、既存ドキュメントを読み取り、
不足情報のみユーザーにヒアリングしながら `CLAUDE.md` を作成します。

## 事前準備（自動読み取り）

以下の順で読み取りを行い、収集できた情報をユーザーに提示してから進める。

### 1. 既存 CLAUDE.md の確認
- `CLAUDE.md` が存在すれば読み込む → **加筆・更新モード**で進める
- 存在しなければ **新規作成モード**で進める

### 2. docs/ 配下のファイルを確認
`docs/` ディレクトリに存在するファイルを一覧取得し、以下を読み込む（存在する場合のみ）:

| ファイル | 読み取る情報 |
|---------|------------|
| `docs/planning.md` | プロジェクト名・概要・技術スタック |
| `docs/requirements.md` | 機能要件のカテゴリ・概要 |
| `docs/basic-functional-design.md` | 機能一覧・画面構成 |
| `docs/architecture.md` | 技術スタック詳細・レイヤー構成・パッケージ構成 |
| `docs/repository-structure.md` | ディレクトリ構成 |
| `docs/development-guidelines.md` | コーディング規約・コミット規約・テスト方針 |

上記以外の `docs/*.md` も存在すれば一覧表示し、ユーザーに Documents テーブルへの追加要否を確認する。

### 3. 読み取り結果の提示

収集できた情報を以下の形でユーザーに提示する:

```
【docs/ から読み取った情報】
- プロジェクト名: {値 または "未確認"}
- プロジェクト概要: {値 または "未確認"}
- 技術スタック: {値 または "未確認"}
- 検出したドキュメント: {ファイル一覧}
- ディレクトリ構成: {値 または "未確認"}
- Conventions（コミット言語等）: {値 または "未確認"}

【ヒアリングが必要な項目】
- {不足している項目のリスト}
```

---

## ヒアリング（不足情報のみ）

docs/ から取得できなかった項目についてのみ確認する。

### A. プロジェクト情報（planning.md から取得できない場合）
- プロジェクト名・一言概要

### B. 技術スタック（architecture.md から取得できない場合）
- 言語・FW・DB・外部API・デプロイ先
- → コードレビュー対象言語リスト（例: `Java (Spring Boot) または Vue.js (TypeScript)`）を決定する

### C. ドキュメント一覧（docs/ に存在しないものがある場合）
- 今後作成予定のドキュメントはあるか（ファイルパス・用途）
- docs/ 以外の場所にドキュメントがあるか

### D. プロジェクトフェーズ構成
docs/ の存在ファイルからフェーズを自動推定して提示し、確認・修正を求める:

```
（例）以下のフェーズを検出しました。修正があれば教えてください:
- [x] プロジェクト初期化
- [x] 企画策定 → docs/planning.md      ← planning.md が存在するため [x]
- [x] 要件定義 → docs/requirements.md  ← requirements.md が存在するため [x]
- [ ] 実装 ← 次のステップ
- [ ] テスト
```

完了済みにすべきフェーズや、追加・削除したいフェーズを確認する。

### E. PMスタイル設定
1. **ユーザーの技術レベル**:
   - 主要言語・FWの経験は？（初心者 / 中級者 / 上級者）
   - 初心者の場合、アノテーション・クラス・メソッドの説明を作業指示に含める

2. **ペアプロ形式の採用有無**:
   - ステップごとにインタラクティブに進めるか？（推奨: Yes）
   - コードはユーザーが書く？（推奨: Yes）

3. **Q&A記録の有無**:
   - 技術的な質問を記録するか？記録先ディレクトリは？（デフォルト: `.questions/`）

### F. Conventions（development-guidelines.md から取得できない場合）
- ドキュメントの記述言語
- コミットメッセージの言語
- コードコメントの言語
- その他プロジェクト固有の規約

---

## 出力

全情報が揃ったら `template.md` の構成で CLAUDE.md を生成し、内容をユーザーに提示する。
確認・修正を受けた後、プロジェクトルートの `CLAUDE.md` として保存する。
