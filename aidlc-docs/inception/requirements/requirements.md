# Requirements Document — Fitbit Weight Management AI Agent

## Intent Analysis

| 項目 | 内容 |
|------|------|
| **ユーザーリクエスト** | Fitbitの健康データを取得し、目標体重達成のためのプランニングを行うAIエージェントを構築する |
| **リクエスト種別** | New Project（Greenfield） |
| **スコープ** | Cross-system（フロントエンド + バックエンド + AIエージェント + 外部API） |
| **複雑度** | Complex（フルスタック構成、OAuth2認証、ReActエージェント、複数API連携） |
| **深度レベル** | Comprehensive |

---

## プロジェクト概要

Fitbit APIから取得した健康データ（歩数・心拍数・体重・摂取カロリー・消費カロリー）をもとに、LLMエージェントがユーザーの目標体重達成に向けた食事プランと自宅運動プランを提案するチャット形式のウェブアプリケーション。

---

## 機能要件（Functional Requirements）

### FR-01: Fitbit OAuth2 認証
- ユーザーがFitbitアカウントでOAuth2認証を行えること
- Authorization Code Flowを実装すること
- アクセストークン・リフレッシュトークンを安全に管理・更新すること
- トークン失効時に自動リフレッシュすること

### FR-02: Fitbit データ取得
エージェントが以下のFitbit APIエンドポイントからデータを取得できること：

| データ種別 | Fitbit API エンドポイント | 備考 |
|-----------|--------------------------|------|
| 歩数 | `GET /1/user/-/activities/steps/date/{date}/1d.json` | 日次集計 |
| 消費カロリー | `GET /1/user/-/activities/calories/date/{date}/1d.json` | アクティビティ由来 |
| 心拍数 | `GET /1/user/-/activities/heart/date/{date}/1d.json` | 安静時心拍含む |
| 体重 | `GET /1/user/-/body/log/weight/date/{date}.json` | ユーザー入力値 |
| 摂取カロリー | `GET /1/user/-/foods/log/caloriesIn/date/{date}/1d.json` | **注記参照** |
| BMI | `GET /1/user/-/body/log/bmi/date/{date}.json` | 体重から算出 |

> **重要注記（摂取カロリー）**: Fitbit の摂取カロリーデータはユーザーが Fitbit アプリで食事を手動ログしている場合のみ取得可能です。エージェントはこのデータを読み取ることはできますが、自動追跡はできません。食事ログが未入力の場合、エージェントはユーザーにその旨を通知し、チャットで直接入力を促すことができます。

### FR-03: AIエージェント（ReAct + NL生成）
- LangGraph を用いた ReAct エージェントを実装すること
- エージェントは以下のツールを自律的に呼び出せること：
  - Fitbit データ取得ツール（各エンドポイントに対応）
  - 目標体重・期間から必要カロリー赤字を計算するツール
  - 運動プラン生成ツール（自宅でできる運動のみ）
  - 食事プラン生成ツール
- LLMが自然言語でサマリー・提案・考察を生成すること
- 会話の文脈を保持したマルチターン対話をサポートすること

### FR-04: 目標体重プランニング機能
- ユーザーが目標体重・目標期間を入力できること
- 現在の体重から目標体重までに必要なカロリー赤字を計算できること
- 摂取カロリー目標（食事制限）を提案できること
- 自宅でできる運動プログラムを提案できること（器具不要・ジム不要）
- 週次・日次の進捗レポートを生成できること

### FR-05: チャット UI
- Next.js + TypeScript によるチャットインターフェース
- ユーザーメッセージの送信と AIレスポンスの表示
- Fitbit 認証状態の表示（接続済み / 未接続）
- レスポンスのストリーミング表示（UX向上のため推奨）

### FR-06: バックエンド API
- FastAPI（Python）による REST API
- エンドポイント：
  - `POST /chat` : チャットメッセージ送信・エージェント実行
  - `GET /auth/fitbit` : OAuth2 認証開始
  - `GET /auth/fitbit/callback` : OAuth2 コールバック
  - `GET /health` : ヘルスチェック
- フロントエンド（Next.js）からの CORS リクエストを許可

## 非機能要件（Non-Functional Requirements）

### NFR-01: 実行環境
- Phase 1: ローカル実行（開発・動作確認）
- Phase 2: クラウドデプロイ（設計完了後に別途検討）

### NFR-02: テスト
- PBT拡張: **有効（Full Enforcement）**
- フレームワーク選定：
  - Python: **Hypothesis**（FastAPI・エージェントロジック）
  - TypeScript: **fast-check**（Next.jsフロントエンド）
- 詳細は PBT ルール（PBT-01 〜 PBT-10）に従う

### NFR-03: セキュリティ
- セキュリティ拡張: **無効**（PoC/プロトタイプフェーズ）
- 最低限の実装として：
  - Fitbit トークンは環境変数または `.env` ファイルで管理
  - トークンをソースコードに直接記述しない

### NFR-04: 保守性
- コードは機能単位でモジュール分割すること
- 環境変数で設定値を管理すること（`.env` + `python-dotenv`）

## ユーザーシナリオ（主要フロー）

### シナリオ 1: 初回セットアップ
1. ユーザーがアプリにアクセス
2. 「Fitbit と連携する」ボタンをクリック
3. Fitbit 認証画面でアクセスを許可
4. アプリに戻り、連携完了メッセージを確認

### シナリオ 2: 目標体重プランの作成
1. ユーザーがチャットに「目標体重60kgに3ヶ月で落としたい」と入力
2. エージェントが Fitbit から現在の体重・活動量データを取得
3. エージェントが必要なカロリー赤字を計算
4. 食事プランと自宅運動プランを自然言語で提案
5. ユーザーが「もう少し運動を減らした提案にして」と調整を依頼
6. エージェントが修正プランを提示

### シナリオ 3: 進捗確認
1. ユーザーが「今週の進捗を教えて」と入力
2. エージェントが今週の歩数・体重・消費カロリーを取得
3. 目標に対する進捗と次週のアドバイスを生成

| 項目 | 内容 |
|------|------|
| 運動プラン | ジム・器具不要の自宅でできる運動のみ |

| 初期フェーズ | ローカル実行のみ（デプロイは後続フェーズ） |
