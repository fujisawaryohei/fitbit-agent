# Output Contract: issue

GitHub Issue 本文として使える形式で出力する。

```
### 種別
feat / fix / chore / docs のいずれか

### タイトル（要約）
やりたいことを 30 文字以内で要約

### 概要
問題・背景を 2〜3 文で

### 仕様
- 具体的な実装内容を箇条書きで

### TODO
- [ ] 実装アクション 1（{ファイルパス} に {何を}）
- [ ] 実装アクション 2
- [ ] テストを追加する

### 参照ドキュメント
- `ファイルパス` — 説明

### 完了条件
- [ ] 検証可能な状態 1（「〜が動作する」「〜が返る」「テストが通る」）
- [ ] 検証可能な状態 2
```

- TODO は実装手順、完了条件は動作確認の観点で分ける。
- 1 チェックボックス = 1 アクション / 1 検証可能な状態。
