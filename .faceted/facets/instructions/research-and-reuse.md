# Instruction: research-and-reuse

新規実装の前に必ず実行する（既存資産の調査と再利用）。

1. **GitHub コード検索を最初に**: `gh search repos` / `gh search code` で既存実装・テンプレート・パターンを探す。
2. **ライブラリドキュメントを次に**: Context7 やベンダー一次ドキュメントで API 挙動・使い方・バージョン差分を確認する。
3. **Exa は上記で不十分なときだけ**: より広い Web リサーチ・探索に使う。
4. **パッケージレジストリを確認**: npm / PyPI / crates.io 等。ユーティリティを自作する前に検索する。
5. **流用可能な実装を探す**: 問題の 80%+ を解く OSS をフォーク・移植・ラップできないか。

→ 自作より、実績ある手法の採用・移植を優先する。
