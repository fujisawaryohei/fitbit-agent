# Knowledge: design-patterns

## Repository パターン
データアクセスを一貫したインターフェースの背後に隠蔽する。
- 標準操作を定義: findAll, findById, create, update, delete
- 具象実装がストレージ詳細（DB / API / ファイル）を扱う
- ビジネスロジックは抽象インターフェースに依存（ストレージ機構に非依存）
- データソースの差し替えとモックによるテストが容易になる

## API レスポンス形式
全レスポンスで一貫したエンベロープを使う。
- 成功/ステータス指標
- データペイロード（エラー時は null）
- エラーメッセージフィールド（成功時は null）
- ページネーション用メタデータ（total, page, limit）
