# Policy: security

## コミット前の必須チェック
- [ ] ハードコードされたシークレット（API キー・パスワード・トークン）がない
- [ ] すべてのユーザー入力を検証している
- [ ] SQL インジェクション対策（パラメータ化クエリ）
- [ ] XSS 対策（HTML サニタイズ）
- [ ] CSRF 対策が有効
- [ ] 認証・認可を検証済み
- [ ] 全エンドポイントにレートリミット
- [ ] エラーメッセージが機密情報を漏らさない

## シークレット管理
- ソースコードにシークレットを書かない。
- 環境変数またはシークレットマネージャを使う。
- 起動時に必須シークレットの存在を検証する。
- 露出した可能性のあるシークレットはローテーションする。

## 対応プロトコル
セキュリティ問題を発見したら: 即停止 → CRITICAL を先に修正 → 露出シークレットをローテーション → 同種の問題をコードベース全体で点検。
