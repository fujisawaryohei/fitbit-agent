# Policy: testing

## 最低カバレッジ: 80%

必須のテスト種別:
1. **Unit** — 個々の関数・ユーティリティ・コンポーネント
2. **Integration** — API エンドポイント・DB 操作
3. **E2E** — 重要なユーザーフロー（言語ごとにフレームワーク選定）

## TDD ワークフロー（必須）
1. テストを先に書く（RED）
2. 実行して失敗を確認する（FAIL）
3. 最小実装を書く（GREEN）
4. 実行してパスを確認する（PASS）
5. リファクタする（IMPROVE）
6. カバレッジ 80%+ を検証する

## 失敗時
- テスト分離を確認する／モックを確認する。
- テストが間違っている場合を除き、テストではなく実装を直す。
