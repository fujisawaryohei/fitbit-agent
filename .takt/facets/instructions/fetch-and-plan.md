タスク入力からチケット番号を特定し、GitHub Issue を取得して実装プランを策定してください。

**手順:**
1. タスク入力（`{task}`）からチケット番号を抽出する
2. `gh issue view <番号> --json title,body,comments` で Issue 本文とコメントを取得する
   - 取得できない場合は理由を報告し、`Issue not found or cannot proceed` を選択する
3. Issue の要求を読解し、要件を分解する（明示要求と、そこから直接導ける暗黙要求に限定）
4. 既存コードを調査し、影響範囲・ファイル構成・実装方針を決める
5. 実装者が迷わないガイドラインを作成する（参照すべき既存実装を file:line で示す）

**Previous Response が存在する場合は、レビューによる差し戻しです。** 指摘を反映して計画を更新してください。

完成したら `Planning complete` を選択してください。
