# Coding Policy

実装時の規約。実装ステップと、実装をレビューするステップの両方で適用する。

## 原則

| 原則 | 基準 |
|------|------|
| Simple > Easy | 書きやすさより読みやすさを優先 |
| DRY | 本質的な重複を排除 |
| イミュータブル | 既存オブジェクトを破壊的変更せず、新しい値を返す |
| コメント | Why のみ。What / How は書かない |
| 関数サイズ | 1 関数 1 責務、~50 行 |
| ファイルサイズ | ~300-400 行を目安、最大 800 行 |
| Fail Fast | エラーは早期に検出。握りつぶさない |

## フォールバック・デフォルト引数の禁止

値の流れを不明瞭にするコードを書かない。

| 禁止パターン | 例 | 問題 |
|---|---|---|
| 必須データのフォールバック | `user?.id or "unknown"` | エラーになるべき状態で処理が継続する |
| try-catch で空値を返す | `except: return ""` | エラーを握りつぶす |
| 不整合値の暗黙スキップ | `if a != expected: return None` | 設定ミスが実行時に無視される |

## 入力検証

- システム境界（API・外部レスポンス・ユーザー入力）で必ず検証する
- スキーマベースの検証を使う。外部データを信頼しない
