# Git / GitHub Hygiene Policy

Issue 操作・コミット・PR 作成に共通する規律。

## 外向き操作の安全性

- コメント投稿・PR 作成・push は、対象（Issue 番号・ブランチ・差分）を確認してから実行する
- コマンドの結果（exit code・出力）を必ず確認する。失敗を無視して次に進まない
- 投稿・作成した URL や番号を成果物に報告する

## コミット / PR

- 1 タスク = 1 PR。無関係な変更を混ぜない
- コミットメッセージは Conventional Commits 形式（`feat:`, `fix:`, `refactor:` など）
- コミット・PR への AI 署名や Co-Authored-By は付けない
- PR 本文は決定事項と実際の差分に忠実に書く。やっていないことを書かない

## 記録

- Issue への記録は、決定済みの事実のみを書く。検討中・未確定の内容を確定として書かない
